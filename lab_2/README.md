# Проект: Анализ телеметрии и прогноз лабораторных измерений

## Описание проекта

Данный проект посвящён анализу производственной телеметрии и построению модели прогнозирования лабораторных измерений на основе временных рядов.

В ноутбуке выполняются следующие этапы:

* разведочный анализ данных (EDA);
* анализ пропусков и выбросов;
* исследование временных лагов;
* корреляционный анализ признаков;
* генерация временных признаков;
* обучение нескольких моделей машинного обучения;
* сравнение качества моделей;
* анализ важности признаков;
* оценка переобучения.

---

## Установка и запуск

### 1. Создать виртуальное окружение (рекомендуется)

```bash
python -m venv venv
```

### 2. Активировать окружение

**Windows**

```bash
venv\Scripts\activate
```

**Linux / macOS**

```bash
source venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Запустить Jupyter Notebook

```bash
jupyter notebook
```

После запуска открыть файл:

```bash
lab_2.ipynb
```

## Интерпретация результатов

## Разведочный анализ (EDA)

На этапе EDA анализируются:

* тренды телеметрии;
* сезонность;
* выбросы;
* пропуски;
* распределение целевой переменной.

Это позволяет понять качество исходных данных перед моделированием.

---

## Анализ временных лагов

В проекте используется временной лаг (`TIME_LAG = 15 минут`), так как лабораторные измерения отражают состояние процесса с задержкой.

Это важно для корректного совмещения телеметрии и целевой переменной.

---

## Построение моделей

Используются три модели:

* Random Forest
* XGBoost
* Ridge Regression

---

## Основные метрики качества

Оцениваются:

* MAE (средняя абсолютная ошибка)
* RMSE (корень из средней квадратичной ошибки)
* MAPE
* WAPE
* Direction Accuracy

---

## Как интерпретировать метрики

* **MAE ниже → лучше**
* **RMSE ниже → лучше**
* **Direction Accuracy выше → лучше**

Если ошибка на тесте значительно выше, чем на обучении — модель переобучена.

---

## Интерпретация важности признаков

Наиболее значимые признаки:

* telemetry_15
* telemetry_14
* telemetry_0

Это означает, что данные датчики сильнее всего влияют на лабораторный показатель.

---



По результатам работы:

* XGBoost показывает лучшую точность;
* Random Forest близок по качеству;
* Ridge уступает нелинейным моделям;
* присутствует переобучение, требующее дальнейшей регуляризации.
